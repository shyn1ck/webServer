- Что такое JSON ?
JSON (JavaScript Object Notation) — это легковесный формат обмена данными, который легко читается и пишется
человеком и легко парсится и генерируется машинами. Он основан на подмножестве языка JavaScript, но является
языконезависимым и может использоваться с различными языками программирования. JSON обычно используется для
передачи данных между сервером и веб-приложением в сети.

Основные элементы JSON

Объекты: Наборы пар "ключ-значение", заключённые в фигурные скобки {}.
{
    "name": "John",
    "age": 30,
    "city": "New York"
}
Массивы: Упорядоченные наборы значений, заключённые в квадратные скобки [].


- Что такое API ?
API (Application Programming Interface) — это набор определений и протоколов, которые позволяют
различным программным
приложениям взаимодействовать друг с другом. API предоставляет методы и инструменты для создания программного
обеспечения и служит посредником, через который одно приложение может использовать функции другого.

Основные концепции API

Эндпоинты: Это URL-адреса, через которые доступен API. Каждый эндпоинт обычно выполняет определённую функцию, например,
получение данных или их отправка.
Методы HTTP: В API, особенно в веб-API, часто используются HTTP-методы для взаимодействия с сервером:
GET: Запрашивает данные с сервера.
POST: Отправляет данные на сервер.
PUT: Обновляет существующие данные на сервере.
DELETE: Удаляет данные с сервера.
Форматы данных: Обычно API использует стандартные форматы данных, такие как JSON или XML, для обмена информацией между
клиентом и сервером.
Аутентификация и авторизация: Многие API требуют, чтобы клиент был аутентифицирован (удостоверён) и авторизован
(получил права на доступ к данным). Это может быть реализовано через API-ключи, токены доступа и другие механизмы
безопасности.
